# 腾讯云短信接入指南

本项目已接入腾讯云短信能力（`TencentSmsImpl`），代码层面可通过后台"系统配置 → 短信平台"切换即可生效。下面是你在腾讯云控制台需要完成的操作清单。

---

## 一、开通短信服务

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)。
2. 搜索并进入 **云短信 SMS**。
3. 首次使用点击"立即开通"，个人实名认证 or 企业认证均可（企业认证签名通过率更高）。
4. 开通后在概览页可以看到免费额度和测试配额。

---

## 二、申请短信签名（SignName）

签名是短信开头的【】内容，必须和你的网站/主体匹配才能通过审核。

路径：`云短信 → 国内短信 → 签名管理 → 创建签名`

填写要点：
- **签名类型**：选"网站"（如果你有已备案的域名，比如 `591jiajiao.cn`）
- **签名内容**：建议填写网站名，比如 `591家教网`
- **证明材料**：上传 ICP 备案截图或网站截图
- **应用场景**：勾选"登录注册"
- **申请说明**：简述用途，例如"用于用户注册/登录/找回密码的身份验证"

⚠️ 审核一般 2 小时～1 个工作日。签名只要一个即可，后面所有验证码/通知都会共用它。

---

## 三、申请短信正文模板（TemplateId）

路径：`云短信 → 国内短信 → 正文模板 → 创建正文模板`

### 模板一：验证码（必填，用于注册/登录/找回密码）

- **模板类型**：验证码类
- **模板名称**：`登录注册验证码`
- **模板内容**：`您的验证码是{1}，{2}分钟内有效，请勿泄露。`
- **应用场景**：验证码

⚠️ 必须使用两个占位符 `{1}` 和 `{2}`，且顺序必须是"验证码 → 有效分钟数"——代码里 `sendVerCode` 传的就是这两个参数（`code, "5"`）。

### 模板二：购买通知（可选，课程购买成功时用）

如果暂时不用，可以先不申请；用时再补即可。

- **模板类型**：通知类
- **模板内容**：`您已成功购买课程{1}，订单号{2}，感谢您的支持。`

模板审核一般 2 小时内。通过后每个模板会分配一个数字 ID（如 `1234567`），就是 **TemplateId**，后面要填到后台。

---

## 四、创建应用，拿到 SdkAppId

路径：`云短信 → 应用管理 → 应用列表 → 创建应用`

- **应用名称**：例如 `591家教网-生产`
- **应用描述**：随意

创建后列表里会有一个 7 位数字的 **SdkAppId**（例如 `1400123456`），后面要填到后台。

---

## 五、创建 API 密钥（SecretId / SecretKey）

路径：`访问管理 CAM → 访问密钥 → API 密钥管理 → 新建密钥`

⚠️ **安全建议**：不要用主账号的密钥，按下面步骤创建一个"子账号"，只给它短信发送权限：

### 推荐方案：子账号 + 最小权限

1. `访问管理 CAM → 用户 → 用户列表 → 新建用户 → 自定义创建`
2. 选"可访问资源并接收消息"
3. 用户名：`sms-sender-591`，访问方式**只勾**"编程访问"（不要控制台登录）
4. 下一步"设置用户权限"，搜索 `QcloudSMSFullAccess` 勾上（只给短信权限）
5. 完成后会显示一次 **SecretId** / **SecretKey**，**立刻保存**（关掉就看不到了）

### 省事方案（不推荐）

直接用主账号的 API 密钥，在 `访问管理 → API 密钥管理` 里创建。功能一样但泄露风险大。

---

## 六、填入网站后台

登录管理后台（`/admin/system/config` 或"系统管理 → 系统配置 → 短信平台"），把上面拿到的值填进去：

| 配置项 key | 填什么 | 示例 |
|-----------|--------|------|
| `smsPlatform` | 平台编号，填 `3` 切换到腾讯云 | `3` |
| `tencentSmsSignName` | 第 2 步审核通过的**签名内容** | `591家教网` |
| `tencentSmsSdkAppId` | 第 4 步拿到的 SdkAppId | `1400123456` |
| `tencentSmsAccessKeyId` | 第 5 步的 SecretId | `AKIDxxxxxxxxxxxxxxxxxxxx` |
| `tencentSmsAccessKeySecret` | 第 5 步的 SecretKey | `xxxxxxxxxxxxxxxxxxxxxxxx` |
| `tencentSmsAuthCode` | 第 3 步"验证码模板"的 TemplateId | `1234567` |
| `tencentSmsPurchaseCode` | 第 3 步"购买通知模板"的 TemplateId（没申请留空） | `1234568` |
| `tencentSmsRegion` | 地域，默认 `ap-guangzhou` | `ap-guangzhou` |

---

## 七、回归测试

1. 修改 `smsPlatform = 3` 后保存配置。
2. 在前端 `/login` 页点"短信登录"，输入手机号 → 点击"发送验证码"。
3. 查后端日志 `/home/ubuntu/logs/user-service.log`：
   - 成功：`验证码已发送: mobile=..., scene=login, platform=腾讯云`
   - 失败：会打印 `腾讯云短信发送失败`，并带有腾讯返回的错误 code。
4. 手机应收到：`【591家教网】您的验证码是 123456，5 分钟内有效，请勿泄露。`

常见错误 code：

| code | 含义 | 处理 |
|------|------|------|
| `FailedOperation.SignatureIncorrectOrUnapproved` | 签名没通过审核或名字不匹配 | 回去控制台看签名状态、把填写的签名内容对一遍 |
| `FailedOperation.TemplateIncorrectOrUnapproved` | 模板未通过 | 同上 |
| `LimitExceeded.PhoneNumberDailyLimit` | 同手机号当日次数超限 | 等第二天，或在后台调整限流策略 |
| `AuthFailure.SecretIdNotFound` / `SignatureFailure` | 密钥错了 | 重新复制 SecretId/Key，注意去除首尾空格 |

---

## 八、回滚与灰度

- **回滚**：把 `smsPlatform` 改回 `5`（MOCK），重新保存配置即可，代码无需改动。
- **灰度**：短信服务的切换完全走配置，和业务代码解耦。生产/预发可以保留在 `5`（MOCK）观察日志，确认稳定后再切 `3`（腾讯云）。

---

## 九、费用与额度

- 国内短信 **0.045 元/条**（新用户首 100 条免费）。
- 开通后默认每日单手机号最多 10 条、每分钟最多 1 条，这两个限制在后台"云短信 → 业务配置 → 发送频率限制"里可以调整。
- 建议保留默认频率限制，防止被恶意刷短信。代码里也已经做了 5 分钟 2 条的 Redis 计数限流（`CODE_STAT`）。
