# 家教在线平台 - P0 移动端设计规范

## 1. 目标与约束

本设计文档用于定义家教在线平台在 `P0` 阶段的移动端门户体验，服务于后续 `Nuxt 3 + Vue 3` 门户前端实现。移动端采用响应式 Web 方案，不单独开发 App。

### 1.1 设计目标

- 让游客可在手机端快速完成浏览教员、查看需求、注册登录。
- 让教员可在手机端完成资料维护、认证上传、查看申请与预约。
- 让学员可在手机端完成资料维护、发布需求、筛选教员、发起预约。
- 保持页面结构轻量，优先保证核心闭环，而不是堆叠复杂功能。

### 1.2 设计约束

- 仅做移动端响应式门户，不扩展为独立 App。
- 本期不接入在线支付，所有支付入口只展示占位提示。
- 联系方式对非 VIP 用户默认脱敏展示。
- 页面结构以现有领域模型、接口字段和 `docs/route-map.md` 为依据。
- 本文档是设计交付，不引入新的云厂商依赖或前端工程耦合。

## 2. P0 范围定义

### 2.1 用户角色

- 游客：浏览首页、教员库、需求库、注册、登录。
- 教员：维护个人资料、上传认证、申请需求、管理预约、查看反馈。
- 学员：维护学员资料、发布需求、处理教员申请、创建预约、管理收藏与足迹。

### 2.2 页面范围

| 模块 | 页面数 | 说明 |
|------|--------|------|
| 公共页面 | 8 | 首页、教员列表、教员详情、需求列表、需求详情、登录、注册、关于我们 |
| 教员中心 | 9 | 首页、资料、编辑资料、认证管理、我的申请、我的预约、预约详情、收到的评价、收藏需求 |
| 学员中心 | 13 | 首页、资料管理、资料编辑、我的需求、发布需求、查看申请、备选列表、我的预约、创建预约、预约详情、我的评价、收藏教员、浏览足迹 |
| 合计 | 30 | 其中关于我们为静态信息页 |

### 2.3 P0 核心闭环

1. 游客浏览首页，进入教员详情或需求详情。
2. 游客注册并选择角色，进入“我的”完成身份初始化。
3. 教员完善资料并提交认证，然后浏览需求并申请。
4. 学员完善资料，发布需求，查看申请与备选，最终发起预约。
5. 预约完成后，双方通过个人中心继续管理反馈、收藏、足迹。

## 3. 信息架构与导航

### 3.1 全局导航结构

移动端底部采用四个一级入口：

- 首页
- 找教员
- 找需求
- 我的

说明：

- 未登录时，“我的”展示登录注册、平台优势、常见问题入口。
- 已登录时，“我的”展示当前身份卡片，并提供“切换为教员视角 / 学员视角”。
- 角色切换不改变底部导航，只改变“我的”首页内容与快捷操作。

### 3.2 页面层级

| 层级 | 页面类型 | 交互要求 |
|------|----------|----------|
| L1 | 首页、找教员、找需求、我的 | 底部 Tab 常驻 |
| L2 | 列表筛选、详情页、中心首页 | 顶部返回，底部 Tab 可按场景隐藏 |
| L3 | 表单编辑、认证上传、创建预约 | 聚焦任务，使用顶部返回 + 底部提交栏 |

### 3.3 导航规则

- 登录拦截：游客点击收藏、预约、发布需求、申请需求时，先跳转登录，完成后返回原页。
- 表单流程：所有编辑页均支持“保存草稿”或“仅保存”。
- 深层页面：优先使用页面返回，不强依赖面包屑。
- 筛选项：统一使用底部抽屉或半屏弹层，不使用桌面式侧边筛选栏。

## 4. 视觉与响应式规范

### 4.1 设计关键词

- 可信：让家长和学员感到正规、清晰、可追踪。
- 温和：避免过于强销售感，突出真实教员与真实需求。
- 高效：在小屏上减少认知跳转，尽量把关键决策放在首屏。

### 4.2 断点规范

| 设备区间 | 宽度 | 设计策略 |
|----------|------|----------|
| Mobile S | 320-359px | 单列布局，文本与按钮紧凑 |
| Mobile M | 360-389px | 设计基线宽度的压缩版 |
| Mobile L | 390-767px | 设计主适配区间 |
| Tablet | 768-1199px | 双列或更宽容器，但保留移动端信息结构 |
| Desktop | 1200px+ | 由后续 PC 设计文档覆盖 |

### 4.3 设计基线

- 主设计稿宽度：`375px`
- 内容最大宽度：`100%`
- 页面左右安全边距：`16px`
- 卡片间距：`12px`
- 区块间距：`20px`
- 吸底操作栏高度：`56px`

## 5. 组件规范

| 组件 | 作用 | 移动端规则 |
|------|------|------------|
| 顶部导航栏 | 页面标题、返回、更多操作 | 高度 `52px`，标题单行居中，右侧仅保留 1 个操作 |
| 搜索栏 | 关键词搜索 | 首页常驻，列表页支持吸顶 |
| 筛选 Chips | 科目/年级/地区/身份筛选 | 超过一行可横向滑动，不换成多列密集按钮 |
| 信息卡片 | 教员卡、需求卡、资料卡 | 卡片主信息不超过 5 行，关键按钮最多 2 个 |
| 状态标签 | 审核、预约、需求状态 | 使用纯色浅底，颜色来自状态令牌 |
| 吸底操作栏 | 收藏、预约、申请、提交 | 主要按钮放右侧，次要动作放左侧 |
| 表单区块 | 编辑资料、发布需求、创建预约 | 按“基础信息/偏好/联系信息”分组 |
| 上传卡片 | 认证材料、反馈图片 | 每张图片单独卡片展示状态与删除操作 |
| 时间轴 | 审核进度、预约进度 | 仅在详情页使用，不在列表页出现 |
| 空状态 | 无收藏、无申请、无足迹 | 提供一条说明 + 一个返回浏览入口 |

## 6. 页面模板设计

### 6.1 首页模板

首页由以下区块组成：

- 顶部欢迎区：品牌名、定位文案、搜索框
- 快捷入口：找教员、找需求、教员入驻、发布需求
- 推荐教员：横向滑动卡片
- 最新需求：纵向卡片列表
- 平台优势：认证审核、需求匹配、客服跟进
- 底部服务信息：联系客服、常见问题、关于我们

### 6.2 列表页模板

适用于教员列表、需求列表、收藏列表、足迹列表。

- 顶部：搜索框 + 筛选按钮
- 中部：筛选 Chips 区
- 主体：列表卡片，首屏 6-8 条
- 底部：加载更多或“已经到底”

### 6.3 详情页模板

适用于教员详情、需求详情、预约详情。

- 顶部：封面信息卡
- 中部：分区块详情
- 底部：固定操作栏

详情页分区顺序：

1. 身份摘要
2. 核心能力或需求概述
3. 时间/价格/区域等决策信息
4. 信任增强信息（认证、案例、流程提示）
5. 行动入口

### 6.4 个人中心首页模板

- 顶部：身份卡片 + 头像 + 状态
- 中部：数据概览卡（申请数、预约数、审核状态、VIP）
- 快捷操作宫格
- 最近动态列表

### 6.5 表单编辑模板

适用于编辑资料、发布需求、创建预约、提交反馈。

- 顶部标题
- 分组表单区块
- 关键字段使用选择器或抽屉，不直接暴露复杂输入
- 底部固定提交栏

### 6.6 列表状态模板

适用于“我的申请”“我的预约”“收到的评价”等个人中心列表。

- 顶部状态筛选
- 列表卡以状态为第一视觉焦点
- 每个卡片只保留一个主动作，其他动作放进详情页

## 8. 关键交互规范

### 8.1 登录与注册

- 注册页第一步先选身份：教员、学员、同时开通。
- 登录支持“验证码登录”作为移动端默认方式，密码登录为次级入口。
- 登录成功后回到触发页，不强制跳转首页。

### 8.2 教员详情页动作

- 主要动作：立即预约
- 次要动作：收藏
- 非登录用户点击动作时弹出登录引导底部弹层
- 非 VIP 用户在详情页只看到脱敏联系方式

### 8.3 需求详情页动作

- 教员视角显示“申请需求”
- 学员本人查看自己的需求时显示“查看申请”“查看备选”
- 非匹配角色隐藏不适合的主按钮，避免操作混乱

### 8.4 资料与认证

- 资料完整度使用进度条展示
- 认证材料按“身份证 / 学历证 / 教师资格证 / 其他”分类
- 审核中状态禁止重复提交，但允许补充材料

### 8.5 预约流程

1. 学员在教员详情页发起预约
2. 填写授课方式、时间、地址和备注
3. 教员收到待确认预约
4. 确认后进入已确认状态
5. 完成后可评价

### 8.6 列表状态

- 加载中：使用卡片骨架屏
- 空状态：说明原因并提供回流按钮
- 错误状态：卡片内重试，不强制整页报错
- 结束态：使用“已经到底了”而不是自动循环加载

## 9. 适配细则

- 所有底部操作栏需适配 `safe-area-inset-bottom`
- 头像、卡片封面优先使用 `3:4` 或 `1:1` 比例，避免高度抖动
- 表单键盘弹起时，提交按钮保持可触达
- 地区、科目、年级等长列表必须使用搜索 + 选择弹层，不在页面内完整展开
- Tablet 端沿用移动端信息架构，但可将列表卡片扩展为双列

## 11. 验收口径

- 游客能在 `3` 次点击内进入教员详情或需求详情。
- 登录注册在手机端单手可操作，主按钮始终可见。
- 教员资料编辑、认证上传、需求发布、预约创建均可在一屏内看清当前进度。
- 列表页首屏能看到至少 `2` 张完整卡片和筛选入口。
- 关键状态有明确视觉区分：待审核、已通过、已拒绝、待确认、已完成、已取消。
- 不出现支付误导文案；若出现支付入口，必须显示“在线支付暂未接入，请联系客服”。
