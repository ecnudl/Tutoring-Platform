# 家教在线平台 - 差距分析报告

## 1. 概述

本文档对现有 roncoo-education 在线教育平台与目标"家教在线平台"进行差距分析，明确可复用模块、需新增模块及需改造模块。

---

## 2. 现有模块（可直接复用）

| 模块 | 说明 | 复用方式 |
|------|------|----------|
| **Users（用户体系）** | 注册、登录、手机验证码、JWT Token | 直接复用，需扩展 userType 字段 |
| **Lecturer（讲师管理）** | 讲师资料、审核流程 | 参考其审核模式，设计教员认证流程 |
| **Region（地区管理）** | 省市区三级联动 | 直接复用，用于教员服务区域、学员所在区域 |
| **RBAC（权限管理）** | 角色、菜单、权限分配 | 直接复用，新增教员/学员角色 |
| **SMS（短信服务）** | 验证码发送、短信模板 | 直接复用，Mock 模式开发 |
| **Upload（文件上传）** | 图片、文件上传至 OSS/本地 | 直接复用，用于证书照片、头像上传 |
| **Orders（订单体系）** | 订单创建、状态流转 | 参考其模式，但本期不接入支付 |

## 4. 需改造模块

### 4.1 Users 表 - userType 字段扩展

**现状：**
- userType 仅区分"普通用户"和"管理员"

**改造方案：**
```
userType 枚举值扩展：
  1 = 学员（Student）
  2 = 教员（Tutor）
  3 = 既是学员又是教员
  9 = 管理员（Admin）
```

**影响范围：**
- 注册接口需支持角色选择
- 登录后 Token 中携带 userType
- Gateway 路由鉴权需识别新角色

### 4.2 Gateway 路由扩展

**现状：** 已有 `/user/api/`、`/user/auth/`、`/user/admin/` 三级路由

**改造：** 在 user 服务下新增家教相关接口路径，无需新增微服务

### 4.3 RBAC 角色扩展

新增角色：
- `ROLE_TUTOR` - 教员角色
- `ROLE_STUDENT` - 学员角色

新增菜单：
- 管理后台：教员审核、需求管理、预约管理、VIP管理、数据字典

## 5. 不涉及改造的模块

| 模块 | 原因 |
|------|------|
| Course（课程） | 家教平台不涉及在线课程 |
| Video（视频） | 不涉及视频点播 |
| Live（直播） | 本期不涉及 |
| Pay（支付） | 本期 VIP 手动开通，不接入支付 |

## 6. 技术风险评估

| 风险项 | 等级 | 应对措施 |
|--------|------|----------|
| Users 表改造影响现有功能 | 中 | userType 新增枚举值，不影响旧值 |
| 数据量增长（足迹表） | 低 | 定期清理超过90天的足迹 |
| 文件上传量（认证材料） | 低 | 复用现有 OSS 配置 |
| 短信发送量 | 低 | 开发环境 Mock，生产环境限流 |

---

## 7. 总结

- **可直接复用：** 7 个模块
- **需新增：** 17 张表，涵盖核心业务和运营支撑
- **需改造：** Users 表字段扩展、RBAC 角色扩展、Gateway 路由配置
- **预计工作量：** 后端 10-15 人天，前端 10-15 人天，联调测试 5 人天
